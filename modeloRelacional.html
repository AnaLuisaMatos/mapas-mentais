Proposto por Codd (anos 70) — modelo lógico baseado em teoria dos conjuntos
Modelo Relacional

Conceitos Básicos

  • Dados armazenados como relações
  • Esquema: R(A1, A2, ..., An)
    • Mapeamento Formal ↔ Informal
    • Relação ↔ Tabela
    • Atributo ↔ Coluna
    • Tupla ↔ Linha
  • Estado da relação: conjunto de tuplas em um dado momento ("fotografia")
  • Domínio: conjunto de valores atômicos válidos de um atributo
    • Grau vs Cardinalidade
    • Grau (aridade): nº de atributos
    • Cardinalidade: nº corrente de tuplas (linhas)
    • ⚠️ Não confundir com cardinalidade de relacionamento (MER)!

Características

    Quantidade de dados
    • Nº de atributos: finito
    • Nº de tuplas: ilimitado (na teoria)
    Ordem dos dados
    • Linhas (tuplas): não ordenadas
    • Colunas: ordem relevante no esquema formal
    • Com mapeamento nome→valor, ordem das colunas não importa
    Valor nulo (NULL)
    • Indica valor desconhecido ou não aplicável

Chaves

    Superchave
    • Subconjunto de atributos que não se repete entre tuplas
    • Pode ter atributos redundantes
    Chave candidata
    • Superchave mínima (sem redundância)
    • Pode haver mais de uma
  • Chave primária: chave candidata escolhida — nunca pode ser NULL
    • Chave estrangeira (FK)
    • Referencia atributo de outra relação (ou da mesma, em auto-relacionamento)
    • Pode ter valores duplicados e NULL

Restrições

  • Implícitas: inerentes ao modelo (grau finito, tuplas únicas...)
  • Explícitas: expressas no esquema via DDL
  • Semânticas: regras de negócio (ASSERTIONS/TRIGGERS)
    • Restrições baseadas em esquema
    • De domínio: valor deve pertencer ao domínio do atributo
    • De chave: unicidade da chave primária
    • Sobre valores nulos: define se atributo aceita NULL
    • Integridade da entidade: PK nunca pode ser NULL
    • Integridade referencial: FK deve ser NULL ou apontar para tupla existente

Operações Básicas

  • Recuperações: vistas em Álgebra Relacional
    • Atualizações
    • Inserir: pode violar qualquer restrição de esquema
    • Excluir: pode violar só integridade referencial
    • Atualizar: depende de qual atributo é alterado (PK, FK ou comum)
    Tratamento de exclusão (FK referenciada)
    • restrict: impede a exclusão
    • cascade: propaga a exclusão
    • set null: define FK como NULL
    • set default: define FK com valor padrão

Esquema × Instância × Catálogo

    Modelo de Dados
    • Conjunto de conceitos, estruturas e restrições para descrever/organizar dados
    • Ex: relacional, hierárquico, orientado a objetos
    Esquema
    • Estrutura lógica / "design" do banco (tabelas, colunas, tipos, restrições)
    • Estático — raramente muda
    • Representa a intenção do banco
    • Representado por diagramas
    Instância (Estado)
    • Conteúdo atual do banco em um dado momento
    • Dinâmica — muda com inserir/excluir/atualizar
    • Também chamada de extensão do esquema
    • Fase final do projeto: banco criado e em operação
    Catálogo
    • Área do SGBD onde ficam os metadados
    • Armazena descrições do esquema (tabelas, colunas, índices) + direitos de acesso
    • Garante que o estado siga as restrições do esquema
    Fluxo: Modelo de Dados → Esquema → Instância. O esquema pode evoluir (evolução de esquema), até com o banco em operação em SGBDs modernos. A PK e demais restrições ficam definidas no esquema (e descritas no catálogo) — mas seus valores aparecem preenchidos na instância.
